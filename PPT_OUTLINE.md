### Slide 2: 什么是蓝眼泪现象?
- **定义**: 海洋浮游生物 (鞭毛藻) 在受到刺激时发出蓝色荧光
- **视觉效果**: 海浪、脚步踩踏时泛起梦幻蓝光
- **观测地点**: La Jolla Shores, San Diego (全美最佳观测点之一)
- **社交媒体热度**:
  - Instagram #lajollabioluminescence: 50K+ posts
  - Reddit r/sandiego: 每年夏季高频讨论
  - YouTube视频观看量: 数百万次

### Slide 3: 商业问题与目标客户

**核心问题**: 在La Jolla开展"蓝眼泪主题夜间观光tour"是否可行？

**目标客户**:
1. **户外运动公司** (如La Jolla Kayak)
   - 已有白天业务 (皮划艇、浮潜)
   - 寻求夜间收入增长点

2. **旅游局/商会**
   - 打造La Jolla夜间旅游品牌
   - 吸引更多游客过夜消费

**市场机会**:
- ✅ 社交媒体UGC内容驱动需求
- ✅ 现有市场空白 (无专门蓝眼泪tour)
- ✅ 差异化竞争优势 (独特夜间体验)

### Slide 4: 主要挑战

**运营挑战**:
- ❌ 蓝眼泪出现不可预测 (环境依赖)
- ❌ 夜间运营成本高 (安全、保险、人力)
- ❌ 淡旺季明显 (夏季高峰 vs 冬季低谷)

**分析目标**:
- 📊 预测哪些夜晚适合观测
- 📊 设计动态定价和排班策略
- 📊 评估不同运营策略的盈利能力

## Part 3: Descriptive Analytics (3分钟)

### Slide 7: 环境数据季节性分析

**时间序列图**:
```
水温 (°C)
20 ┤     ╭──────╮
18 ┤   ╭─╯      ╰─╮
16 ┤  ╭╯          ╰╮
14 ┼──╯            ╰──
   └─1─3─5─7─9─11──月份
```

**关键发现**:
- 夏季 (6-8月): 水温18-20°C, 最适合蓝眼泪
- 春秋季 (4-5月, 9-10月): 中等概率
- 冬季 (11-3月): 低概率 (水温<15°C)

### Slide 8: 月相对观测质量的影响

**月相周期图**:
```
观测评分
10 ┤ ●           ●
 8 ┤  ╲         ╱
 6 ┤   ╲       ╱
 4 ┤    ╲  ○  ╱
 2 ┤     ╲   ╱
 0 ┼──────●───────
   新月  上弦 满月 下弦
```

**洞察**:
- 新月期 (月相0-20%): ⭐⭐⭐⭐⭐ (月光最弱, 最佳)
- 上弦/下弦: ⭐⭐⭐ (中等)
- 满月期: ⭐ (月光太亮, 不推荐)

### Slide 9: Descriptive Analytics总结

**关键模式**:
1. **最佳季节**: 夏季 (6-8月) 评分平均7.5/10
2. **最佳月相**: 新月±5天 评分平均8.2/10
3. **周末效应**: 周末需求比工作日高40%
4. **时段偏好**: 晚上8-10点最受欢迎

## Part 4: Predictive Analytics (3分钟)

### Slide 10: 机器学习模型设计

**模型选择**: Random Forest Regressor

**为什么选Random Forest?**
- ✅ 非线性关系建模能力强
- ✅ 对异常值和噪声鲁棒
- ✅ 特征重要性可解释
- ✅ 无需大量手动调参

**输入特征** (4个):
- `water_temp`: 海水温度 (14-20°C)
- `wave_height`: 浪高 (0.5-3.0m)
- `wind_speed`: 风速 (2-10 m/s)
- `moon_phase`: 月相 (0-1, 0=新月)

**输出**: 观测评分 (0-1, 转换为0-10星级)

### Slide 11: 模型性能评估

**测试集表现**:
```
指标          值
─────────────────
RMSE        0.052
MAE         0.041
R²          0.847
```

**特征重要性排名**:
```

2. moon_phase   ████████████   28%
3. wave_height  ████████       22%
4. wind_speed   ████           15%
```

**解释**: 水温和月相是最关键因素

## Part 5: Transaction Data Analytics (3分钟)

### Slide 13: 交易数据设计

**多场次设计** (更贴近真实业务):
- 每天0-3场tour (根据预测评分动态决定)
- 时段: 20:00, 21:00, 22:00

**多维度建模**:
```
渠道 (Channel):
├── Website直订 (50%) - 成本最低, 利润率最高
├── OTA平台 (30%)    - 佣金20%, 客流量大
└── Walk-in (20%)    - 转化率低, 无佣金

产品类型 (Product):
├── Standard ($40-$60) - 1.5小时, 基础装备
└── Premium ($60-$80)  - 2小时, 摄影服务, 小吃
```

### Slide 14: Polars高性能分析

**为什么用Polars?**
- ⚡ 比Pandas快5-10倍
- 🧠 更高效的内存使用
- 📊 语法简洁 (类似SQL)

**示例代码**:
```python
monthly_perf = (
    transactions_pl
    .group_by(["month", "product_type"])
    .agg([
        pl.col("revenue").sum(),
        pl.col("profit").sum(),
        pl.col("num_customers").sum(),
    ])
)
```
